Introducción a la maquetación.

Elementos semánticos

  • HEADER Es el contenido introductorio que contiene los títulos principales, a veces suele tener una galeria a modo de presentación y también la navegación.
    • NAV Es la navegación principal y donde se encuentran solamente los links mas importantes de nuestro sitio.
  • MAIN Es el contenido principal donde colocamos la información relevante para el usuario, la mayoría de los elementos se encontrarán adentro.
    • SECTION Elemento que forma parte del contenido, depende del contexto y solamente funciona dentro de la página, ya que está ligado al contenido de la misma.
    • ARTICLE Elemento independiente al contexto, funciona tanto dentro como fuera de la página y debe ser autoconclusivo.
    • ASIDE Elemento lateral o contenido tangencial que guarda una relación indirecta con sus elementos hermanos.
  • FOOTER Es el contenido de desenlace, podemos encontrar información del autor, datos y/o formularios de contacto, mapas del sitio e incluso links a las redes sociales.

Jerarquia de selectores CSS

  • Tag selector: es aquel que apunta a un tipo de etiqueta, si bien se pueden combinar es el menos especifico y por lo tanto al ser muy general con respecto a los demas, es el que posee menor prioridad ya que no permite distinguir entre las etiquetas que seleccionamos.
    • header{color:darkred}
    • header h1{color:tomato}
  • Class selector: es aquel que apunta a las clases aplicadas sobre los elementos y se diferencia por la implementacion de un punto (.) antes del nombre de la clase. Especificamente permite seleccionar un grupo de etiquetas, incluso pudiento aclarar que tipo de etiqueta es a la que apuntamos.
    • .header{color:darkgreen}
    • .header h1{color:limegreen}
  • ID selector: es aquel que apunta al Id de un elemento en particular, se diferencia por la implementacion de un numeral (#). Es el selector con mayor prioridad porque se considera muy especifico con respecto a los demas, la razon de esto es que no deberian existir mas de una etiqueta con el mismo Id.
    • #header{color:darkblue}
    • #header h1{color:skyblue}

Atributos generales

  • Box-sizing:
    • content-box: final-width = width + padding + border | content-width = width;
    • border-box: final-width = width | content-width = width - padding - border;
  • Display:
    • BLOCK: ocupa todo su contenedor, genera salto de línea (no comparte espacio), se les puede definir tamaño (contenedores);
    • INLINE: ocupa el ancho de su contenido, comparte espacio con otras etiquetas, su tamaño es automatico - ancho (contenido);
    • INLINE-BLOCK: ocupa el espacio de su contenido, comparte espacios, se les puede definir tamaño - ancho/alto (contenido);
Para más información consultar Parrot

Flexbox

Es una herramienta que permite indicar a un contenedor la forma en la que se van a distribuir y organizar sus elementos a diferencia de los valores vistos anteriormente que corresponden al atributo display y modifican la forma en la que se representa el elemento seleccionado, flexbox actua sobre la etiqueta padre y afecta a las etiquetas que sean hijos directos.

  • Flex-start: Es el valor predeterminado y permite distribuir los elementos al principio del flex (izquierda).
  • Flex-end: Permite distribuir los eleementos en orden hacia el final del flex (derecha).
  • Center: Alínea todos los elementos hacia el centro del flex (horizontal).
  • Space-around: Calcula la distancia que debe tener cada elemento y la aplica al flex. Las distancias entre los elementos que se encuentran en el medio es mayor debido a que estas se suman.
  • Space-between: Separa de extremo a extremo los elementos dentro del flex y la distancia resultante es equitativa entre cada etiqueta, teniendo en cuenta que el primer y último elemento están en el límite de la caja del contenido.
  • Space-evenly: Calcula exactamente la misma distancia de separación para cada etiqueta, quedando distribuido de manera precisa, pero no es compatible con todos los navegadores.

Align-items

  • Flex-start: A diferencia del atributo justify-content, no hay valor predeterminado y permite organizar a los elementos al principio del flex (arriba).
  • Flex-end: Organiza los elementos dentro del flex hacia el final del mismo. Es necesario que el contenedor posea un alto definido.
  • Center: Organiza los elementos hacia el centro de la fila (verticalmente).
  • Baseline: Acomoda las etiquetas independientemente de su tamaño de manera tal que el texto quede alineado para mejor legibilidad
  • Align-self: Atributo que permite a la etiqueta seleccionada dentro del flex cambiar su alineación con respecto a los demás elementos.

Pseudo

  • :Pseudo-clases

    Estado o caracteristica de un elemento sobre el cual podemos trabajar para editar la etiqueta en cuestion.
    • :first-child - Primer hijo de un contenedor
    • :hover - Cuando pasamos el mouse sobre un elemento
    • :focus - Cuando seleccionamos y estamos sobre el elemento (a / input / textarea / select / button)
    • :link - Hipervínculo sin visitar
    • :visited - Hipervínculo visitado
    • :active - Momento de selección del hipervínculo
    • :last-child - Último hijo de un contenedor
    • :first-of-type
    • :last-of-type

  • ::Pseudo-Elementos

    Parte interna de un elemento a la que se le puede aplicar estilos independientemente de su contenedor.
    • ::first-line - Primera línea de un elemento
    • ::first-letter - Primera letra de un elemento
    • ::before - Contenido antes del contenido
    • ::after - Contenido despues del contenido

  • Grid-template-columns: Define el ancho de las columnas de nuestra grilla, así como también la cantidad de las mismas mediante la repetición de valores, por defecto al aplicar CSS-Grid se utiliza una única columna.
  • Grid-template-rows: Define el alto de las filas especificadas, a diferencia de grid-template-columns donde cada valor representa una columna, aquí simplemente definimos el tamaño pero no la cantidad, por defecto este es auto.
  • Gap: Separación entre filas y columnas respectivamente, es independiente al margin de los elementos internos y posee dos versiones individuales del atributo row-gap y column-gap.
  • Grid-column: Atajo que permite indicar a un elemento hijo su posición, simplificando los atributo grid-column-start / grid-column-end tomando en cuenta que cada línea imaginaria a la izquierda representa el comienzo de las mismas.
  • Grid-row: Atajo que permite indicar a un elemento hijo su posición, simplificando los atributo grid-row-start / grid-row-end tomando en cuenta que cada línea imaginaria desde arriba representa el comienzo de las mismas.
